# 리저렉션(Resurrection) PRD

## 1) 문서 개요
- **문서명**: Product Requirements Document (PRD) - 리저렉션
- **버전**: v1.0 (초안)
- **작성일**: 2026-03-12
- **대상 제품**: Windows용 듀얼 모니터 창 배치 복원 앱
- **관련 코드베이스**: `SProj.Resurrection`

## 2) 배경 및 문제 정의
듀얼 모니터/외부 모니터 환경에서는 모니터 연결 상태가 바뀌는 순간, 사용자가 배치해둔 창 위치와 크기가 쉽게 무너진다.  
특히 개발자, 디자이너, 사무 사용자처럼 다수의 앱 창을 동시에 사용하는 경우, 매번 수동 재배치에 많은 시간이 소요된다.

리저렉션은 현재 열려 있는 창들의 위치/크기를 레이아웃으로 저장하고, 필요 시 즉시 복원함으로써 반복적인 재배치 비용을 줄이는 것을 목표로 한다.

## 3) 제품 목표
### 3.1 비즈니스/제품 목표
- 모니터 변경 상황에서 사용자의 작업 복구 시간을 단축한다.
- 단순하고 빠른 복원 경험을 제공해 "창 배치 복원"을 일상 워크플로우로 만든다.
- 트레이 상주 기반의 저마찰 실행 경험으로 지속 사용률을 높인다.

### 3.2 사용자 목표
- 자주 쓰는 창 배치를 이름으로 저장하고 재사용하고 싶다.
- 모니터 연결/해제 후에도 이전 작업 환경을 빠르게 되찾고 싶다.
- 복원 전 확인 여부, 자동 복원 여부를 내 취향대로 제어하고 싶다.

## 4) 범위
### 4.1 포함 범위 (In Scope)
- 현재 열려 있는 창들의 위치/크기 캡처
- 레이아웃 저장/목록 조회/복원/삭제
- 현재 창 목록 미리보기
- 모니터 변경 이벤트 감지 후 마지막 레이아웃 자동 복원(옵션)
- 복원 전 확인 다이얼로그(옵션)
- 트레이 상주 및 앱 창 숨김/다시 열기
- 로컬 영구 저장(`electron-store`)

### 4.2 제외 범위 (Out of Scope)
- 실행 중이지 않은 앱을 자동 실행해 복원하는 기능
- 계정 기반 클라우드 동기화
- 팀 단위 공유 레이아웃
- 창 매칭의 고급 규칙(프로세스/클래스 기반 정밀 매칭)
- macOS/Linux 배포 지원(현 단계는 Windows 배포 기준)

## 5) 대상 사용자 및 핵심 시나리오
### 5.1 주요 사용자군
- **개발자**: IDE, 터미널, 브라우저, 문서 툴을 다중 모니터에 고정 배치
- **디자이너/영상 편집자**: 툴 패널과 작업 창을 정해진 위치로 유지
- **사무 사용자**: 메신저, 문서, 브라우저, 그룹웨어를 반복 배치

### 5.2 대표 사용자 시나리오
1. 사용자가 "작업용" 레이아웃을 저장한다.
2. 노트북 도킹/언도킹으로 모니터 구성이 바뀐다.
3. 앱이 모니터 변경을 감지하고(자동 복원 ON 시) 복원을 제안/실행한다.
4. 사용자는 수초 내 기존 작업 배치를 회복한다.

## 6) 기능 요구사항
### FR-01 레이아웃 저장
- 사용자는 레이아웃 이름을 입력해 현재 창 배치를 저장할 수 있어야 한다.
- 저장 시 보이는 창(유효한 크기/제목)을 수집해야 한다.
- 저장 결과에는 창 개수와 생성 시각이 포함되어야 한다.

**수용 기준**
- 빈 이름 입력 시 저장이 차단되고 오류 메시지가 표시된다.
- 저장 성공 시 목록에 즉시 반영된다.

### FR-02 레이아웃 목록 조회
- 저장된 레이아웃 목록을 최신순 또는 저장순으로 표시해야 한다.
- 각 항목은 이름, 창 개수, 저장 시각을 보여야 한다.

**수용 기준**
- 앱 재시작 후에도 저장된 목록이 유지된다.
- 마지막 복원 레이아웃이 시각적으로 구분된다.

### FR-03 레이아웃 복원
- 사용자는 특정 레이아웃을 선택해 복원할 수 있어야 한다.
- 복원 결과(성공/실패, 복원된 창 수)를 피드백해야 한다.

**수용 기준**
- 복원 성공 시 상태 메시지가 표시된다.
- 대상 레이아웃이 없으면 오류를 반환한다.

### FR-04 레이아웃 삭제
- 사용자는 저장된 레이아웃을 삭제할 수 있어야 한다.
- 삭제된 레이아웃이 마지막 복원 대상이면 참조를 초기화해야 한다.

### FR-05 자동 복원
- 모니터 추가/제거/해상도 변경 이벤트를 감지해야 한다.
- 설정값에 따라 마지막 복원 레이아웃을 자동 적용해야 한다.

**수용 기준**
- `autoRestore = true`이고 `lastLayoutId`가 있을 때만 동작한다.
- 모니터 이벤트 직후 일정 안정화 대기 후 복원을 시도한다.

### FR-06 복원 전 확인
- 사용자는 복원 전에 확인 다이얼로그를 표시할지 설정할 수 있어야 한다.
- 수동 복원과 자동 복원 모두 설정을 따라야 한다.

### FR-07 현재 창 목록 미리보기
- 사용자는 현재 캡처 가능한 창 목록을 조회할 수 있어야 한다.
- 창 제목, 프로세스명, 좌표/크기를 확인할 수 있어야 한다.

### FR-08 트레이 상주 동작
- 앱 창 닫기 시 앱이 종료되지 않고 트레이로 숨김 처리되어야 한다.
- 트레이에서 앱 다시 열기/종료가 가능해야 한다.

## 7) 비기능 요구사항
### NFR-01 성능
- 레이아웃 저장/목록 조회/복원 트리거는 사용자 체감 지연이 낮아야 한다.
- 일반적인 사용 환경에서 저장/복원 요청의 UI 반응은 즉각적이어야 한다.

### NFR-02 안정성
- 네이티브 모듈(`node-window-manager`) 로드 실패 시 앱 전체가 종료되지 않아야 한다.
- 창 캡처/복원 실패는 기능 단위로 격리되고 오류 로그를 남겨야 한다.

### NFR-03 보안
- 렌더러에서 Node 직접 접근을 차단하고 IPC 브리지만 사용해야 한다.
- 민감 정보는 수집하지 않으며, 로컬 저장 데이터는 레이아웃/설정에 한정한다.

### NFR-04 호환성
- 배포 대상은 Windows x64를 우선 지원한다.
- 설치형 배포(NSIS) 기준으로 설치/업데이트 흐름을 제공한다.

### NFR-05 사용성
- 초보 사용자도 1분 내 "저장-복원" 핵심 플로우를 이해할 수 있어야 한다.
- 상태 메시지와 확인 다이얼로그는 한국어 중심의 명확한 문구를 제공한다.

## 8) 데이터 모델 요구사항
### 8.1 Layout
- `id`: 문자열(유니크)
- `name`: 사용자 입력 이름
- `windows`: 창 정보 배열
- `createdAt`: 타임스탬프

### 8.2 WindowInfo
- `processName`, `title`, `x`, `y`, `width`, `height`

### 8.3 Settings
- `autoRestore`: 모니터 변경 시 자동 복원 여부
- `askBeforeRestore`: 복원 전 확인 여부
- `lastLayoutId`: 마지막 복원 레이아웃 ID

## 9) 성공 지표 (KPI)
- **복원 성공률**: 복원 시도 대비 1개 이상 창 복원 성공 비율
- **시간 절감 체감**: 사용자 설문 기준 "수동 재배치 대비 시간 단축" 응답 비율
- **반복 사용률**: 주간 2회 이상 레이아웃 복원 사용자 비율
- **안정성**: 세션당 치명적 오류(앱 비정상 종료) 발생률

## 10) 리스크 및 대응
- **창 매칭 정확도 리스크**: 제목 기반 매칭은 동명이창/제목 변경에 취약  
  → 향후 프로세스 경로/핸들/규칙 기반 매칭 보강 검토
- **네이티브 모듈 의존 리스크**: 환경별 로드 실패 가능성  
  → 로드 실패 시 기능 비활성화 + 사용자 안내 + 로그 강화
- **모니터 이벤트 타이밍 리스크**: 연결 직후 복원 시점이 불안정할 수 있음  
  → 안정화 대기 및 재시도 정책(향후) 검토

## 11) 릴리스 계획 (권장)
### Phase 1 - 기본 가치 제공 (완료/유지)
- 레이아웃 저장/복원/삭제
- 트레이 상주
- 설정(자동 복원/확인 다이얼로그)

### Phase 2 - 신뢰성 고도화
- 복원 실패 상세 원인 분류
- 창 매칭 보강(제목 + 프로세스 규칙)
- 실패 케이스 재시도/부분 복원 전략

### Phase 3 - 확장
- 레이아웃 태그/검색
- 백업/내보내기
- 다국어/접근성 개선

## 12) 오픈 이슈
- 복원 대상 창 식별 키를 어디까지 확장할지(정확도 vs 복잡도)
- 자동 복원 시 사용자 제어 수준(즉시/확인/스케줄 지연) 세분화 여부
- 향후 멀티 플랫폼 전략 필요 여부
